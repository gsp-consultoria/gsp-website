# Documentação

Esta pasta concentra documentação orientada a Spec Driven Design para a landing page da GSP Consultoria.

## Como usar

1. Comece por `docs/specs/product-requirements.md`.
2. Leia `docs/specs/technical-spec.md`.
3. Consulte a spec da seção em `docs/specs/sections/`.
4. Use prompts em `.codex/prompts/` para guiar implementação com Codex.
5. Use regras em `.codex/rules/` como contrato de qualidade.
6. Antes de abrir PR, revise `docs/checklists/pr-checklist.md`.
